# Extraction-of-the-bloodline-curse-model-integration-of-the-tool-process
The tools used for extracting the bloodline curse model are too scattered and the extraction process is very cumbersome. This is an integrated workflow that enables direct conversion of dcx files into fbx files. It will also include the tools used and their links, and if there is any infringement, please let me know so I can delete them.

# The tools used:（It merely utilized its software functions without modifying any code, did not generate any profits, and was fully made public.）
Yabber：（Yabber is a tool used for unpacking. It mainly converts dcx files into flver files.）

https://github.com/JKAnderson/Yabber?tab=readme-ov-file#readme

BloodBorne models：（Convert the file to smd and ascll，I haven't been able to trace the origin of this tool; I only found this post.）

https://www.zenhax.com/viewtopic.php@t=7006.html

SMD to FBX：（The conversion of SMD to FBX utilized binary encoding.）

This tool was written by the "corsor" tool.

# Approach/Strategy：
Utilizing the simulation dragging function, move the files in sequence to achieve automation and directly convert from dcx to fbx.
